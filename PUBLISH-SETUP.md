# Публікація змін з адмінки прямо на сайт — що зроблено і що налаштувати

## Що змінилось
Раніше кожна зміна в адмінці (товар, категорія, відгук) вимагала: натиснути
«Зберегти *.json» → завантажити файл → вручну замінити його на хостингу →
перезібрати сайт. Тепер це відбувається одним кліком:

1. Ви редагуєте товари/категорії/відгуки в адмінці як завжди.
2. Натискаєте кнопку **«🚀 Опублікувати на сайті»** у відповідній вкладці.
3. Дані одразу зберігаються на сервері (Netlify Blobs — те саме сховище, де вже
   зберігаються замовлення), а сайт автоматично перезбирається.
4. Приблизно за **~30–90 секунд** зміни з'являються на сайті для всіх відвідувачів —
   і на головній сторінці, і на окремих сторінках товарів/категорій (вони й далі
   генеруються заздалегідь, тому Google як і раніше бачить окрему сторінку на
   кожен товар — SEO не постраждало).

Кнопки «Резервна копія (.json)» і «Копіювати JSON» нікуди не зникли — це просто
зручний спосіб зробити копію собі на комп'ютер про всяк випадок, для роботи
сайту вони більше не обов'язкові.

## Що обов'язково потрібно налаштувати в Netlify

### 1. Створити Build Hook
Це посилання, натискання на яке (POST-запит) каже Netlify «перезбери сайт
зараз». Адмінка викликає його автоматично після кожної публікації.

1. Зайдіть у **Site configuration → Build & deploy → Build hooks** (або
   «Site settings → Build & deploy» → розділ «Build hooks» — залежно від версії
   інтерфейсу).
2. Натисніть **Add build hook**, дайте назву (наприклад `admin-publish`),
   оберіть гілку (зазвичай `main`), натисніть **Save**.
3. Скопіюйте посилання виду `https://api.netlify.com/build_hooks/xxxxxxxxxxxx`.

### 2. Додати його як змінну середовища
1. **Site configuration → Environment variables → Add a variable**.
2. Ключ: **`BUILD_HOOK_URL`**, значення — посилання з кроку 1.
3. Збережіть.

### 3. Задеплоїти сайт ще раз
Змінні середовища й нова функція `netlify/functions/catalog.js` підхоплюються
лише після нового деплою:
**Deploys → Trigger deploy → Deploy site**.

## Якщо `BUILD_HOOK_URL` ще не задано
Публікація все одно працює — дані збережуться на сервері, просто адмінка
покаже повідомлення «Збережено, але автоматичне оновлення сайту не
запустилось» і сайт доведеться перезібрати вручну (Deploys → Trigger deploy)
або через `git push`, якщо сайт підключений до GitHub.

## Одне уточнення щодо кількох фото на товар
Кожне фото автоматично стискається до JPG (~1000px, ~60-100 КБ). Якщо у товару
багато фото (5-10+), файл `products.json` стає суттєво більшим — на швидкість
сайту для відвідувачів це не впливає (фото й так вантажаться окремо браузером),
але варто уникати десятків фото на один товар без потреби, щоб публікація й
завантаження в адмінці лишались швидкими.

## Як це влаштовано технічно (коротко)
- Дані пишуться в Netlify Blobs через нову функцію `netlify/functions/catalog.js`
  (запис — тільки для залогінених адмінів, той самий захист токен-сесією, що й
  для замовлень).
- `build.py` перед кожною збіркою сам підтягує актуальні дані з цього сховища
  і перезаписує ними `products.json` / `categories.json` / `reviews.json` —
  і вже з них генерує сторінки товарів/категорій, sitemap тощо.
- Якщо з адмінки ще нічого не публікували (наприклад, одразу після
  встановлення цього оновлення) — `build.py` просто продовжує користуватись
  файлами, які лежать у репозиторії, як і раніше. Нічого не ламається.

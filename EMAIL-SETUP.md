# Email-листи клієнту про замовлення — налаштування Resend

## Що зроблено
Якщо клієнт вказав **email** при оформленні замовлення, йому автоматично надсилаються
три листи (кожен — окремо, на відповідному етапі):

1. **Замовлення прийнято** — одразу після оформлення. У листі — фото товарів
   (якщо в каталозі для них задано фото), кількість, сума, спосіб доставки й оплати.
2. **Замовлення відправлено** — коли менеджер в адмінці вписує номер **ТТН**.
   Статус замовлення при цьому автоматично змінюється на «В дорозі». У листі —
   номер ТТН для відстеження.
3. **Просимо залишити відгук** — коли менеджер в адмінці змінює статус замовлення
   на «Виконано». Лист із проханням оцінити магазин і посиланням на сторінку
   відгуків надсилається лише один раз на замовлення (повторна зміна статусу
   листа не дублює).

Якщо email не вказано, або лист з якоїсь причини не надішлеться —
саме замовлення все одно зберігається як завжди й потрапляє в адмінку.
Це ніяк не може «зламати» оформлення чи обробку замовлення.

Листи надсилаються через **Resend** (resend.com) — простий сервіс транзакційної
пошти з безкоштовним тарифом (100 листів/добу, 3000/місяць).

## Що обов'язково потрібно зробити

### 1. Зареєструватись у Resend і підтвердити свій домен
Це важливо: **без підтвердженого домену Resend дозволяє надсилати листи лише на
вашу власну адресу** (для тесту) — клієнтам листи не підуть. Тому:

1. Зареєструйтесь на **resend.com** (можна через Google-акаунт).
2. У боковому меню відкрийте **Domains → Add Domain**, вкажіть `smartstoreua.com`
   (ваш домен, без `https://` і без `www`).
3. Resend покаже кілька DNS-записів (зазвичай TXT для SPF, CNAME/TXT для DKIM,
   іноді MX). Додайте їх у налаштуваннях DNS вашого домену (там же, де
   прив'язаний домен до Netlify — реєстратор домену або Netlify DNS).
4. Зачекайте підтвердження (зазвичай кілька хвилин, іноді до кількох годин) —
   статус домену в Resend стане «Verified».

### 2. Створити API-ключ
1. У Resend: **API Keys → Create API Key**.
2. Права можна лишити «Sending access» (мінімально необхідні).
3. Скопіюйте ключ (виду `re_xxxxxxxxxxxxxxxxx`) — він показується лише один раз.

### 3. Додати змінні середовища в Netlify
**Site configuration → Environment variables → Add a variable**:

| Ключ | Значення |
|---|---|
| `RESEND_API_KEY` | ключ із кроку 2 (`re_...`) |
| `NOTIFY_FROM_EMAIL` | адреса відправника з вашого підтвердженого домену, напр. `SMART STORE <замовлення@smartstoreua.com>` |

Адреса в `NOTIFY_FROM_EMAIL` обов'язково має бути на домені, який ви підтвердили
в кроці 1 (наприклад `замовлення@smartstoreua.com`, `noreply@smartstoreua.com` —
будь-яка адреса на цьому домені підійде, реєструвати окрему поштову скриньку не треба).

### 4. Задеплоїти сайт заново
**Deploys → Trigger deploy → Deploy site**.

## Як перевірити
1. Оформіть тестове замовлення на сайті, вказавши свій email.
2. Перевірте пошту (і папку «Спам» на всяк випадок) — лист має прийти протягом
   кількох секунд.
3. Якщо лист не прийшов — гляньте **Netlify → Functions → orders → Logs**, або
   в Resend-панелі розділ **Emails**, там видно статус кожного надісланого листа
   і причину помилки, якщо вона є.

## Якщо `RESEND_API_KEY` / `NOTIFY_FROM_EMAIL` ще не задані
Нічого не зламається — форма продовжує працювати, замовлення зберігаються,
просто листи клієнтам не надсилаються (поле `emailSent: false` у відповіді
функції). Менеджер і далі бачить нові замовлення в адмінці як завжди.
